PROJECT OVERVIEW
-----------------
SyncRoom is a real-time watch-party web app.
- Frontend: React + Vite (in folder: sync-room-ui/)
- Backend: Node.js + Express + Socket.IO (in folder: sync-room-node/)
- Database: MongoDB Atlas
- Real-time: Socket.IO

The project is a monorepo with this structure:
  sync-room/
    sync-room-node/     (backend)
    sync-room-ui/       (frontend)
    docker-compose.yml
    render.yaml
    README.md

HOW DOCKER WORKS:
  docker compose up --build

  This starts:
  1. Backend container (Node.js on port 8000, internal only)
  2. Frontend container (nginx on port 80, public)

  nginx acts as a reverse proxy:
  - http://localhost/          → serves React app (static files)
  - http://localhost/api/*     → proxies to backend:8000/api/*
  - http://localhost/socket.io → proxies to backend:8000/socket.io (WebSocket)

  Because nginx handles routing, VITE_API_URL is left EMPTY in Docker.
  The frontend uses relative URLs (/api/...) which nginx routes to the backend.

IMPORTANT ABOUT VITE_API_URL:
  - Docker: leave EMPTY (nginx proxies everything)
  - Vercel+Render: must be set to https://your-backend.onrender.com
  - Local dev: set to http://localhost:8000

PHASE 6.5 — CI/CD (GITHUB ACTIONS)
================================================================

WHAT WAS DONE:
- Created two GitHub Actions workflow files
- CI runs on every push and every pull request
- Deploy runs automatically after CI passes on the main branch

WORKFLOW 1: .github/workflows/ci.yml
  Triggers: push to main/develop, pull request to main

  Job 1 — Frontend CI:
    1. Checkout code
    2. Set up Node.js 20
    3. npm ci (uses package-lock.json)
    4. npm run lint (ESLint)
    5. npm test (Vitest — 116 tests)
    6. npm run build (Vite build with VITE_API_URL empty)

  Job 2 — Backend CI:
    1. Checkout code
    2. Set up Node.js 20
    3. npm ci --omit=dev
    4. Syntax check all JS files (node --check)

WORKFLOW 2: .github/workflows/deploy.yml
  Triggers: after the CI workflow completes on main branch

  Job 1 — Deploy Frontend to Vercel:
    (Vercel builds it using env vars set in Vercel dashboard)
    1. Install Vercel CLI globally
    2. vercel --prod (Vercel does the build with VITE_API_URL from its dashboard)

  Job 2 — Deploy Backend to Render:
    1. curl POST to Render deploy hook URL
    2. Fails the job if HTTP response is not 2xx

GITHUB SECRETS REQUIRED:
  VERCEL_TOKEN         — API token from vercel.com/account/tokens
  VERCEL_ORG_ID        — from .vercel/project.json after running "vercel link"
  VERCEL_PROJECT_ID    — from .vercel/project.json after running "vercel link"
  RENDER_DEPLOY_HOOK_URL — from Render dashboard → service → Settings → Deploy Hook

HOW TO DEPLOY MANUALLY (WITHOUT CI/CD)
================================================================

STEP 1 — SET UP MONGODB ATLAS
  1. Go to cloud.mongodb.com and create a free account
  2. Create a new cluster (free M0 tier is fine)
  3. Create a database user: Security → Database Access → Add New User
     Username: syncroom_user, Password: (generate a strong one)
     Role: Atlas Admin (or readWriteAnyDatabase)
  4. Allow network access: Security → Network Access → Add IP Address → Allow from anywhere (0.0.0.0/0)
     (Render uses dynamic IPs so you need to allow all)
  5. Get connection string: Connect → Drivers → copy the URI
     It looks like: mongodb+srv://syncroom_user:PASSWORD@cluster.mongodb.net/sync_db?retryWrites=true&w=majority
  6. Save this URI — you will need it in step 2

STEP 2 — DEPLOY BACKEND ON RENDER
  1. Go to dashboard.render.com and create a free account
  2. New → Web Service
  3. Connect your GitHub account and select the sync-room repository
  4. Settings:
       Name: syncroom-api (or whatever you want)
       Root Directory: sync-room-node
       Runtime: Node
       Build Command: npm ci --omit=dev
       Start Command: node src/server.js
       Branch: main
  5. Add Environment Variables (click "Advanced" or go to Environment tab):
       NODE_ENV = production
       LOG_LEVEL = warn
       PORT = 8000
       MONGO_URI = (paste your Atlas URI from step 1)
       CLIENT_URL = (leave blank for now, fill in after step 3)
  6. Click "Create Web Service"
  7. Wait for it to deploy. Your backend URL will be something like:
       https://syncroom-api.onrender.com
  8. Save this URL — you need it in step 3

STEP 3 — DEPLOY FRONTEND ON VERCEL
  1. Go to vercel.com and create a free account
  2. New Project → Import Git Repository → select sync-room
  3. Configure project:
       Root Directory: sync-room-ui
       Framework Preset: Vite (auto-detected)
       Build Command: npm run build (auto-filled)
       Output Directory: dist (auto-filled)
  4. Add Environment Variables:
       VITE_API_URL = https://syncroom-api.onrender.com
       (this is the Render URL from step 2)
  5. Click Deploy
  6. Wait for it to build and deploy. Your frontend URL will be something like:
       https://syncroom.vercel.app
  7. Save this URL — you need it in the next step

STEP 4 — UPDATE BACKEND CORS
  1. Go back to Render dashboard → syncroom-api → Environment
  2. Update CLIENT_URL to your Vercel URL:
       CLIENT_URL = https://syncroom.vercel.app
  3. Render will automatically redeploy
  4. Wait for it to finish

STEP 5 — TEST EVERYTHING
  1. Open your Vercel URL in two different browsers
  2. Create a room in browser 1
  3. Copy the room code and join in browser 2
  4. Test: chat, playback controls, reactions
  5. Check browser console for errors (F12 → Console)

  Common issues:
  - CORS error → CLIENT_URL on Render is wrong (should match Vercel URL exactly, no trailing slash)
  - Socket fails → VITE_API_URL on Vercel is wrong (should match Render URL exactly)
  - 502 error → Render cold start (wait 30 seconds and try again — free tier sleeps)
  - "Missing env var" → MONGO_URI or CLIENT_URL not set on Render

ENVIRONMENT VARIABLES REFERENCE
================================================================

BACKEND (set on Render):
  MONGO_URI    = mongodb+srv://user:pass@cluster.mongodb.net/sync_db?retryWrites=true&w=majority
  CLIENT_URL   = https://your-frontend.vercel.app    (no trailing slash)
  PORT         = 8000
  NODE_ENV     = production
  LOG_LEVEL    = warn

FRONTEND (set on Vercel):
  VITE_API_URL = https://your-backend.onrender.com   (no trailing slash)

FOR LOCAL DEVELOPMENT:
  Backend .env:
    PORT=8000
    CLIENT_URL=http://localhost:5173
    MONGO_URI=mongodb+srv://...
    NODE_ENV=development
    LOG_LEVEL=debug

  Frontend .env:
    VITE_API_URL=http://localhost:8000

FOR DOCKER:
  Backend .env (same as local)
  VITE_API_URL should be EMPTY or NOT SET
  (nginx proxies /api and /socket.io to the backend container automatically)
